원격 데스크톱(RDP) &mdash; 인증 오류: 요청한 함수가 지원되지 않습니다(CredSSP 오류)
티스토리🗓 2025-09-28

원격 데스크톱 접속 시 아래와 같은 오류 메시지 뜬 적 있나요?

인증 오류가 발생했습니다. 요청한 함수가 지원되지 않습니다.
CredSSP 암호화 오라클을 수정 때문일 수 있습니다.

최근 윈도우 보안 업데이트(또는 패치 미적용) 때문에 클라이언트와 서버 간 CredSSP(자격 증명 보안 공급자) 버전이 맞지 않아서 생기는 문제입니다. 수정 방법을 빠르게 정리합니다 — 긴급 임시방편부터 권장(안전) 방법까지 그대로 붙여넣어 쓰실 수 있게 정리했어요.

원인 요약

  • 클라이언트/서버 중 한쪽만 보안 패치가 되어 있어 CredSSP 규약 불일치 발생
  • 그룹 정책 또는 레지스트리에서 암호화 오라클 정책이 엄격하게 설정되어 있음

해결 방법 (우선순위로 정리)

1) 권장 — 서버/클라이언트 모두 최신 업데이트 적용

가장 안전한 방법입니다. Windows Update에서 최신 보안 패치를 모두 설치한 뒤 재부팅하고 RDP 재시도하세요.

2) 임시: 클라이언트에서 정책 낮추기 (긴급 접속용)

보안 위험이 있으니 회사/중요 시스템이면 사용 전 관리자와 상의하세요.

  1. Win + R → gpedit.msc 실행
  2. 컴퓨터 구성 → 관리 템플릿 → 시스템 → 자격 증명 위임 이동
  3. 암호화 오라클 수정(Encryption Oracle Remediation) 더블클릭
  4. 설정을 사용(Enabled) 으로 바꾸고 보호 수준(Protection Level) 을 취약(Vulnerable) 로 선택
  5. gpupdate /force 실행 후 RDP 재접속

3) 레지스트리로 변경 (정책 편집 불가한 경우)

  1. Win + R → regedit 실행
  2. 경로:
 
HKEY_LOCAL_MACHINE\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\CredSSP\Parameters
  1. AllowEncryptionOracle (DWORD) 값 생성/수정:
  • 0 = 기본(차단)
  • 1 = 호환(경고)
  • 2 = 취약(허용) ← 긴급용

변경 후 재부팅 또는 정책 갱신 후 RDP 재시도.

4) CMD(관리자)로 레지스트리 추가

긴급하게 원격접속을 열어야 할 때 가장 빠른 방법입니다. 아래 명령을 관리자 권한으로 실행하고 PC를 재부팅하세요.

  • 관리자 CMD 실행 후 입력
 
reg add "HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Policies\System\CredSSP\Parameters" /v AllowEncryptionOracle /t REG_DWORD /d 2 /f
 
  • 재부팅
    변경 사항 적용을 위해 재부팅하세요. 이 설정은 AllowEncryptionOracle=2 로 설정하여 취약 모드(허용)로 만듭니다.

보안 주의사항

  • 취약 또는 AllowEncryptionOracle=2 설정은 자격 증명 탈취 위험을 증가시킵니다. 임시로만 사용하고, 가능한 빨리 서버/클라이언트 패치로 복구하세요.
  • 회사 환경이라면 보안팀/시스템 담당자에게 먼저 보고 후 진행하세요.

문제 재발 방지 팁

  • 정기적으로 Windows 보안 업데이트 적용(WSUS 또는 중앙관리)
  • RDP 사용 시 네트워크 레벨 인증(NLA) 적용 및 VPN 내부망 통해서만 접속
  • 원격 데스크톱 게이트웨이(RD Gateway) 도입 고려

마무리

긴급하게 접속해야 한다면 CMD로 레지스트리 추가 → 재부팅 방법으로 빠르게 해결할 수 있습니다.
하지만 근본적 해법은 패치 적용이니, 접속 확보 후 바로 보안 복구 작업(업데이트 및 레지리셋)을 진행하세요.
