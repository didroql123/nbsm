Godot 개발 - 노드 이해하기
티스토리🗓 2025-02-21

Godot 엔진의 핵심 개념 중 하나는 바로 노드(Node)입니다. Godot에서 모든 것은 노드로 이루어져 있으며, 게임을 개발할 때 노드를 효과적으로 사용하는 것이 매우 중요합니다. 이번 글에서는 노드의 개념과 종류, 그리고 어떻게 활용하는지를 살펴보겠습니다.

1. 노드(Node)란?

Godot에서 **노드(Node)**는 게임 내의 개별적인 요소를 나타냅니다. 예를 들어, 캐릭터, 카메라, UI 요소, 사운드, 애니메이션 등 모든 것이 노드로 표현됩니다. 노드는 계층 구조(Tree Structure)를 이루며, 부모-자식 관계를 형성할 수 있습니다.

2. 씬(Scene)과 노드 트리(Node Tree)

Godot에서는 하나의 씬(Scene)이 여러 개의 노드로 이루어져 있습니다.

  • 씬은 특정한 기능을 가진 노드들의 집합입니다.
  • 씬은 재사용이 가능하며, 다른 씬의 일부로 포함될 수도 있습니다.
  • 씬의 최상위 노드는 **루트 노드(Root Node)**라고 하며, 그 아래에 여러 개의 자식 노드들이 연결됩니다.

3. 주요 노드 종류

Godot에는 다양한 유형의 노드가 있으며, 대표적인 노드는 다음과 같습니다.

3.1 일반 노드

  • Node: 기본적인 노드이며, 논리적인 그룹을 만들 때 사용됩니다.

3.2 2D 관련 노드

  • Node2D: 2D 공간에서 위치, 회전, 크기를 조절할 수 있는 노드
  • Sprite2D: 2D 이미지(스프라이트)를 표시하는 노드
  • AnimatedSprite2D: 여러 개의 프레임을 이용해 애니메이션을 구현하는 노드
  • CollisionShape2D: 충돌 감지를 위한 형태(shape)를 정의하는 노드

3.3 3D 관련 노드

  • Node3D: 3D 공간에서 위치, 회전, 크기를 조절할 수 있는 노드
  • MeshInstance3D: 3D 모델을 표시하는 노드
  • Camera3D: 3D 환경을 렌더링하는 카메라 노드

3.4 UI 관련 노드

  • Control: UI 요소를 만들기 위한 기본 노드
  • Button: 버튼을 생성하는 노드
  • Label: 텍스트를 표시하는 노드
  • Panel: UI 컨테이너 역할을 하는 노드

4. 노드 계층 구조 활용하기

노드는 계층 구조(Tree)를 이루므로, 자식 노드는 부모 노드의 영향을 받습니다. 예를 들어:

Character (Node2D)
 ├── Sprite2D
 ├── CollisionShape2D
 └── AnimationPlayer

위의 구조에서 Sprite2D는 캐릭터의 스프라이트를 담당하고, CollisionShape2D는 충돌 감지를, AnimationPlayer는 애니메이션을 관리합니다.

5. 노드 추가 및 스크립트 연결하기

Godot에서 노드를 추가하는 방법은 간단합니다.

  1. Scene 창에서 우클릭 → Add Child Node
  2. 원하는 노드를 검색하여 추가
  3. 스크립트를 추가하려면 노드를 선택한 후 Attach Script 버튼 클릭

예제: Sprite2D에 스크립트 추가하기

extends Sprite2D

func _ready():
    print("Sprite 노드가 준비되었습니다!")

6. 노드 활용 팁

  • 최상위 노드는 되도록 명확한 역할을 가진 노드를 선택
  • 재사용 가능한 씬을 만들어서 유지보수를 쉽게 관리
  • 노드 이름을 직관적으로 설정하여 가독성 향상

결론

Godot에서 노드는 게임을 구성하는 핵심 요소이며, 노드의 구조를 이해하고 효과적으로 활용하면 개발 속도를 높이고 유지보수를 쉽게 할 수 있습니다. 다음 글에서는 노드 간의 신호(Signal)와 스크립트 연동 방법에 대해 다뤄보겠습니다!
