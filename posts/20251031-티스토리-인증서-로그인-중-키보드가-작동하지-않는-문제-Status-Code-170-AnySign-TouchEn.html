인증서 로그인 중 키보드가 작동하지 않는 문제 (Status Code 170, AnySign + TouchEn)
티스토리🗓 2025-10-31

은행이나 공공기관 사이트에서 인증서 로그인을 시도할 때
갑자기 키보드가 입력되지 않거나 완전히 멈추는 현상이 발생하는 경우가 있습니다.
화면에는 아래와 같은 메시지가 표시됩니다.

 
디버그 프로그램을 발견했습니다. (Status Code: 170)

이후에는 키보드 입력이 차단되어
비밀번호나 인증서 암호를 입력할 수 없으며,
브라우저를 꺼도 증상이 지속되는 경우가 많습니다.

1. 문제 증상 요약

  • 인증서 로그인 도중 키보드 입력이 불가능
  • “디버그 프로그램을 발견했습니다” 메시지 출력
  • 공동인증서 창(AnySign)이 멈추거나 비밀번호 입력 불가
  • 브라우저를 종료해도 입력 불가 상태 지속
  • 재부팅 시에만 정상 복구되는 경우 존재

이 문제는 TouchEn nxKey(키보드 보안 프로그램) 과
AnySign(공동인증서 모듈) 이 동시에 실행될 때 발생하는
보안 후킹 충돌 현상입니다.

2. 원인 분석

보안 프로그램은 해커가 비밀번호를 가로채지 못하도록
후킹(Hooking) 기술을 이용해 키보드 입력을 가로채 암호화합니다.

입력 흐름:
키보드 → TouchEn nxKey(암호화) → 브라우저

하지만 AnySign 또한 동일한 방식으로 입력을 후킹합니다.
이 두 프로그램이 동시에 작동하면 TouchEn이 AnySign을
‘디버깅 프로그램’으로 오인하여 보호모드(입력 차단 모드) 로 전환됩니다.
그 결과 Status Code 170 오류가 발생하며 키보드 입력이 완전히 멈춥니다.

3. 해결 방법

(1) 오류창 닫기

창을 닫지 않으면 입력이 계속 차단된 상태가 됩니다.

(2) 작업 관리자에서 프로세스 종료

  • 단축키: Ctrl + Shift + Esc
  • 아래 항목을 찾아 작업 끝내기
    • TouchEn
    • nxKey
    • AnySign
    • KeyCrypt

(3) 브라우저 완전 종료

모든 브라우저(Chrome, Edge, Whale 등)를 닫고
작업 관리자에서 관련 프로세스(chrome.exe, msedge.exe)가 남아 있다면 종료합니다.

(4) 재실행 또는 재부팅

브라우저 재시작 후 입력이 복구되지 않는다면, PC를 재부팅합니다.

※ 임시 조치: Windows + Ctrl + O 단축키를 눌러
화면 키보드(가상 키보드) 를 통해 입력 가능.

4. 재발 방지 방안

항목내용
한쪽만 실행 인증서 로그인 시엔 AnySign만, 일반 로그인 시엔 nxKey만 사용
백신 예외 등록 V3, FortiClient, DLP 등 보안프로그램에서 TouchEn 및 AnySign 폴더를 예외 처리
재설치 제어판 → 프로그램 추가/제거 → 두 모듈 삭제 후 사이트 재접속 시 재설치
원격환경 주의 RemoteView, RDP 등 원격 접속 환경에서 충돌 빈도 높음

5. 참고 경로

  • TouchEn nxKey:
    C:\Program Files (x86)\SoftCamp\TouchEn nxKey\
  • AnySign for PC:
    C:\Program Files (x86)\Wizvera\AnySign4PC\

6. 정리

Status Code 170 오류는
보안 프로그램이 키보드 입력을 보호하는 과정에서
서로의 동작을 ‘디버깅’으로 인식하여 발생하는 오탐입니다.

해결 요약:
오류창 닫기 → 프로세스 종료 → 브라우저 재시작 → 필요 시 재부팅

최근 보안 정책이 강화되면서 이러한 충돌이 증가하고 있습니다.
보안이 중요하지만, 과도한 중복 보안은 시스템 불편을 초래할 수 있습니다.
