Windows 사진 앱 실행 불가 현상 &ndash; 금융 보안 모듈 충돌 원인 분석
티스토리🗓 2025-10-01

최근 Windows 환경에서 기본 사진 앱(Photos) 이 실행되지 않는 현상을 확인했습니다.
일반적으로 앱 손상이나 캐시 문제로 오해하기 쉽지만, 이번 사례는 외부 보안 모듈 충돌이 주요 원인이었습니다.

1. 증상 (Symptoms)

  • JPEG, PNG 등 이미지 파일 더블 클릭 시 앱이 실행되지 않음
  • 작업 관리자(Task Manager)에서는 PhotosApp.exe 프로세스가 잠시 생성 후 즉시 종료
  • 동일 파일을 다른 뷰어(예: IrfanView, Paint)에서 열 경우 정상 동작

즉, 특정 앱(Photos)만 차단되는 상태가 발생했습니다.

2. 원인 분석 (Root Cause)

작업 관리자를 통해 백그라운드 프로세스를 확인한 결과, 다수의 금융/보안 모듈이 상주 중이었습니다.

주요 보안 모듈

  • TouchEn nxKey / nxWeb / nxWeb Policy : 키보드 입력 암호화 및 웹 보안 모듈
  • ImageSAFER : 화면 캡처 방지 모듈
  • VPWallet Service : 전자지갑 및 결제 모듈
  • WIZVERA Delfino : 전자서명 및 결제 보안 모듈
  • nProtect Online Security, AhnLab Safe Transaction : 금융 보안 통합 모듈

이 중 특히 TouchEn 계열 프로그램은 화면 캡처 차단 및 입력 후킹 기능을 수행하며,
해당 과정에서 Windows UWP 앱(예: Photos) 실행을 가로막는 경우가 발생합니다.

3. 해결 방법 (Resolution)

본 사례에서는 TouchEn 모듈 제거 이후 사진 앱이 정상 실행되었습니다.

단계별 조치

  1. 작업 관리자 확인
    • 백그라운드에서 과도한 메모리를 점유하는 금융 보안 모듈 확인
    • (예: TouchEn nxKey, VPWallet, ImageSAFER 등)
  2. 프로그램 제거
    • 제어판 → 프로그램 및 기능 → TouchEn 계열 삭제
    • 삭제 이후 Photos 앱 즉시 정상 동작 확인
  3. 대체 방안
    • 금융 사이트 이용 시, TouchEn 등은 접속 시 자동으로 재설치 가능
    • 필요 시에만 설치하여 사용하고, 일상적인 PC 사용 환경에서는 제거 권장

4. 결론 (Conclusion)

Windows 기본 앱 실행 불가 현상은 단순히 앱 손상이 아니라,
외부 보안 모듈 충돌에서 비롯될 수 있습니다.

특히 TouchEn, ImageSAFER, WIZVERA, VPWallet 등은
보안 강화를 목적으로 설치되지만, 일반 앱의 정상 실행을 방해할 가능성이 존재합니다.

따라서,

  • 업무/개인 PC에서는 불필요한 보안 모듈 제거
  • 금융/관공서 사이트 이용 시에만 재설치
    를 통해 안정적인 시스템 운영이 가능합니다.
